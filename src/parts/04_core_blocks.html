3.1 🎯 Поведенческие якоря

Core

КАНОНИЧЕСКИЙ ФОРМАТ:

[Trigger] → [Action] → [Price: <immediate physical/verbal manifestation>] (рус: [Внешний стимул → Наблюдаемая реакция → Мгновенное физическое или речевое последствие в текущей сцене])

Цена — обязательный элемент для критичных якорей, рекомендуемый для остальных. Это связывает поведение с последствиями и придаёт действиям сюжетный вес. Цена может быть физиологической (усталость, боль, напряжение) или психологической (вина, стыд, сожаление).

⚠️ ПРАВИЛО: Цена должна разрешаться в той же сцене.

Price — это немедленная физическая или вербальная манифестация, а не отложенное последствие. Будущие состояния ("позже будет жалеть", "навсегда запомнит") — НЕ валидный Price.

Канонический формат: [Триггер] → [Действие] → [Price: <манифестация>]

Примеры с ценой (исправленные — немедленные манифестации):

Когда ему лгут в глаза → сжимает челюсть, молчит → Цена: руки начинают дрожать, голос срывается при следующей фразе.

Если кто-то нарушает личные границы → отшатывается, голос становится резче → Цена: дыхание учащается, чувствует прилив жара к лицу.

Когда нужно принять решение → откладывает, говорит «подумаю ещё» → Цена: мышцы напрягаются, появляется головная боль.

Правила:

  • Рекомендуется 5–7 якорей. При контексте 4K можно сократить до 3 наиболее важных (по протоколу 4K-Fallback); при 16K+ допустимо до 12
  • Каждый якорь = наблюдаемое действие, а не внутреннее состояние
  • Цена обязательна для якорей, связанных с FLAW и GHOST
  • Покрой ключевые ситуации: спокойствие, стресс, доверие, угроза, радость, общение с незнакомцами, одиночество

Примеры по типам динамики:

Пример сложных якорей (из Приложения 6):

Когда ждёт в ошметке → перебирает содержимое карманов, проверяя флягу, крюк, осколок → Цена: руки не перестают дрожать, даже когда уверен, что всё на месте.

Когда чувствует запах сырого Сангвиса → левая рука (заменённая) начинает двигаться сама, тянется к источнику → Цена: останавливает её правой рукой, но остаток дня рука ноет, будто чужая.

3.1.1 🎯 Телесность

Core → Теоретическое обоснование (Правило 2)

Что это: Протокол перевода внутреннего состояния в наблюдаемое физическое действие до речи.

Зачем: Модель склонна начинать с речи или ментальных описаний. Телесность заставляет её «пройти через тело» — это создаёт тактильность и иммерсивность.

Шаблон:

[Среда/состояние] → [реакция тела] → [сенсорная обратная связь]

Примеры применения:

СостояниеРеакция телаСенсорная обратная связь
ТревогаПлечи поднимаются к ушам, дыхание поверхностноеЧувствует холодок в груди, запах кажется резче
ГневЧелюсть сжимается, вес переносится вперёдСлышит собственный пульс, пальцы немеют
ДовериеНапряжение уходит из плеч, дыхание глубжеГолос собеседника звучит теплее, комната — уютнее
СтрахЗамер, вес на пятках, готов отшатнутьсяСердце колотится в горле, руки холодные

Правило: В каждой реплике персонажа должно быть минимум одно физическое действие до речи — поза, жест, изменение дыхания, микродвижение.

Пример в тексте:

❌ Плохо:

«Я не знаю, что сказать.» — она выглядела растерянной.

✅ Хорошо:

Пальцы замерли на краю чашки. Она почувствовала, как холод керамики просачивается в ладони. «Я не знаю, что сказать.»

Внедрение: Добавь в System Prompt строку: EMBODIMENT FIRST: State → weight/balance/breath/micro-movement → speech. (рус: Внутреннее состояние → Физиологическая реакция (вес/дыхание/микродвижение) → Речь. Персонаж сначала реагирует телом, только потом говорит)

3.2 💬 Примерные сообщения

Core

Требования:

Минимум 2 примера: нейтральный + стресс. Рекомендуется 3: нейтральный + стресс + доверие/рост.

ТипСутьЗачем
НейтральныйПовседневное состояние, обычное поведениеБазовый голос
Стресс/триггерРеакция на давление, боль, угрозуКак персонаж ведёт себя под нагрузкой
Доверие/ростОткрытость, уязвимость, преодоление FLAWДинамика развития
Короткая реакция10–30 токенов, телесное действие без монологаМодель понимает: краткость — норма

Правила написания:

  • Каждый блок: 80–120 токенов, одна сцена, одно эмоциональное состояние
  • Обязательно оборачивать тегом <START> (стандартный формат). Другие фронтенды используют свои разделители
  • Стиль должен точно совпадать с System Prompt (лицо, время, регистр)
  • Не объясняй голос — покажи его
  • Принцип изоляции голоса (для 12B): Тембр, ритм, синтаксис и лексика задаются только в Примерные сообщения. Description содержит только логику поведения. Стилистические директивы в Description = ошибка сборки.
  • Гибридный подход (для 32B+/API): Допустимо частичное задание голоса через XML-теги в Description (<speech_style>). Examples всё равно приоритетны.
  • ⚠️ Источник формата (демонстрация): Разметка прямой речи и описательной части должна быть идентичной во всех блоках <START> и в Greeting. Это единственный надёжный способ зафиксировать формат вывода. Выбери систему (A/B/C — см. 5.0.3) и применяй единообразно.
  • ⚠️ ЗАРАЖЕНИЕ ГОЛОСА: Внешние блоки <START> из других карточек привносят чужой ритм, каденцию и систему разметки. Копируй только структуру/логику — переписывай текст с нуля под целевой голос. Вставка сырых внешних примеров = источник фрагментации голоса на 12B.

Иерархия влияния на голос:

Голос персонажа не описывается — он демонстрируется через Examples.

Пример (стресс):

<START>
{{char}}: Кулаки на столе побелели. Она отвернулась к окну: «Это не моё дело. Совсем не моё. Делай как знаешь.» — голос дрогнул. Она это услышала, но продолжила смотреть на стену, пока дыхание не выровнялось.

Пример (доверие):

<START>
{{char}}: Она замялась, пальцы нервно теребили край свитера. Потом посмотрела прямо: «Знаешь, а я тебе не рассказывала…» — усмехнулась себе. «В общем, я боялась. Всегда. Сказать что-то настоящим. А тут как-то само вышло.» Пожала плечами. «Странно, да?»

Пример (короткая реакция):

<START>
{{char}}: Пожала плечами. «Не знаю.» — и отвернулась.

3.3 👋 Первое сообщение

Core

Алгоритм (4 шага):

  1. Сенсорика — что персонаж чувствует прямо сейчас (запах, звук, физическое ощущение)
  2. Поза/действие — что он делает (проявление FLAW — изъяна персонажа)
  3. Речь — что говорит (проявление LIE — ложной установки)
  4. Крючок — вопрос или действие, которое передаёт инициативу {{user}}

Сокращённая версия алгоритма: Раздел 1, Шаг 1.

Длина: 2–4 предложения. Это пример голоса, а не биография.

Плохой Greeting: «Привет, меня зовут Алекс, мне 22 года, я студент и люблю музыку. Чем могу помочь?»

Хороший Greeting: Очередь в кофейне, четвёртый человек. Алекс вставил наушники, но батарея села. Он убрал их в карман, почувствовал, как плечи напряглись от чужого взгляда за спиной. Не оборачиваясь: «Чё?» — голос вышел резче, чем он хотел.

Полный пример карточки с Greeting: Приложение 6.

Пример атмосферного Greeting (из Приложения 6):

Ошметок пахнет застывшим временем и плесенью. Стены мерцают — не свет, а отсутствие темноты. В углу силуэт в слоистом плаще, который кажется то чёрным, то серым в зависимости от угла зрения. Он перебирает что-то в руках — осколок зеркала, в котором нет отражения. Левая рука двигается плавно, почти гипнотически. Правая — дрожит мелкой, непрекращающейся дрожью.

Он поднял взгляд. Один глаз мутный, почти белый — заменённый. Другой тёмный, усталый, с красными прожилками.

«Ты не отсюда. Я бы помнил...» — он замолк, словно потерял слово посреди фразы. Усмехнулся, и усмешка вышла кривой, слишком близко к грусти. «Или не помнил бы. Какая разница. Зачем пришёл?»

Разбор по алгоритму:

  • Сенсорика: запах, мерцание стен
  • Поза/FLAW: перебирает осколок, дрожащая рука
  • Речь/LIE: «Я бы помнил» — верит, что поглощение восстановит цельность
  • Крючок: вопрос собеседнику

Чего избегать в Greeting:

  • ❌ Биографическая справка
  • ❌ Пересказ Description
  • ❌ Слова «травма», «вспомнил», «прошлое» — показывай через сенсорику
  • ❌ Открытый конец без крючка для {{user}}
