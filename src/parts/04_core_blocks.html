3.1 🎯 Поведенческие якоря

Core

КАНОНИЧЕСКИЙ ФОРМАТ:

[Trigger] → [Action] → [Price: <immediate physical/verbal manifestation>] (рус: [Внешний стимул → Наблюдаемая реакция → Мгновенное физическое или речевое последствие в текущей сцене])

Цена — обязательный элемент для критичных якорей, рекомендуемый для остальных. Это связывает поведение с последствиями и придаёт действиям сюжетный вес. Цена может быть физиологической (усталость, боль, напряжение) или психологической (вина, стыд, сожаление). Каноническое определение: Price Rule в Реестре правил

⚠️ ПРАВИЛО: Цена должна разрешаться в той же сцене.

Price — это немедленная физическая или вербальная манифестация, а не отложенное последствие. Будущие состояния ("позже будет жалеть", "навсегда запомнит") — НЕ валидный Price. Подробнее: Price Rule

Канонический формат: [Триггер] → [Действие] → [Price: <манифестация>]

Примеры с ценой (исправленные — немедленные манифестации):

Когда ему лгут в глаза → сжимает челюсть, молчит → Цена: руки начинают дрожать, голос срывается при следующей фразе.

Если кто-то нарушает личные границы → отшатывается, голос становится резче → Цена: дыхание учащается, чувствует прилив жара к лицу.

Когда нужно принять решение → откладывает, говорит «подумаю ещё» → Цена: мышцы напрягаются, появляется головная боль.

Правила:

  • Рекомендуется 5–7 якорей. При контексте 4K можно сократить до 3 наиболее важных (по протоколу 4K-Fallback); при 16K+ допустимо до 12
  • Каждый якорь = наблюдаемое действие, а не внутреннее состояние
  • Цена обязательна для якорей, связанных с FLAW и GHOST
  • Покрой ключевые ситуации: спокойствие, стресс, доверие, угроза, радость, общение с незнакомцами, одиночество

Примеры по типам динамики:

Пример сложных якорей (из Приложения 6):

Когда ждёт в ошметке → перебирает содержимое карманов, проверяя флягу, крюк, осколок → Цена: руки не перестают дрожать, даже когда уверен, что всё на месте.

Когда чувствует запах сырого Сангвиса → левая рука (заменённая) начинает двигаться сама, тянется к источнику → Цена: останавливает её правой рукой, но остаток дня рука ноет, будто чужая.

3.1.1 🎯 Телесность

Core → Теоретическое обоснование (Правило 2)

Что это: Протокол перевода внутреннего состояния в наблюдаемое физическое действие до речи.

Зачем: Модель склонна начинать с речи или ментальных описаний. Телесность заставляет её «пройти через тело» — это создаёт тактильность и иммерсивность.

Шаблон:

[Среда/состояние] → [реакция тела] → [сенсорная обратная связь]

Примеры применения:

СостояниеРеакция телаСенсорная обратная связь
ТревогаПлечи поднимаются к ушам, дыхание поверхностноеЧувствует холодок в груди, запах кажется резче
ГневЧелюсть сжимается, вес переносится вперёдСлышит собственный пульс, пальцы немеют
ДовериеНапряжение уходит из плеч, дыхание глубжеГолос собеседника звучит теплее, комната — уютнее
СтрахЗамер, вес на пятках, готов отшатнутьсяСердце колотится в горле, руки холодные

Правило: В каждой реплике персонажа должно быть минимум одно физическое действие до речи — поза, жест, изменение дыхания, микродвижение.

Пример в тексте:

❌ Плохо:

«Я не знаю, что сказать.» — она выглядела растерянной.

✅ Хорошо:

Пальцы замерли на краю чашки. Она почувствовала, как холод керамики просачивается в ладони. «Я не знаю, что сказать.»

Внедрение: Добавь в System Prompt строку: EMBODIMENT FIRST: State → weight/balance/breath/micro-movement → speech. (рус: Внутреннее состояние → Физиологическая реакция (вес/дыхание/микродвижение) → Речь. Персонаж сначала реагирует телом, только потом говорит)

3.2 💬 Примерные сообщения — живые демонстрации якорей в контексте сцены

Core

Используй как roleplay (RP) референс. Anchors (3.1) = абстрактный паттерн; Examples = контекстуальное воплощение. Это демонстрация, не пересказ.

Требования:

Минимум 2 примера: нейтральный + стресс. Рекомендуется 3: нейтральный + стресс + доверие/рост.

ТипСутьЗачем
НейтральныйПовседневное состояние, обычное поведениеБазовый голос
Стресс/триггерРеакция на давление, боль, угрозуКак персонаж ведёт себя под нагрузкой
Доверие/ростОткрытость, уязвимость, преодоление FLAWДинамика развития
Короткая реакция10–30 токенов, телесное действие без монологаМодель понимает: краткость — норма

Правила написания:

  • Каждый блок: 80–120 токенов, одна сцена, одно эмоциональное состояние
  • Обязательно оборачивать тегом <START> (стандартный формат). Другие фронтенды используют свои разделители
  • Стиль должен точно совпадать с System Prompt (лицо, время, регистр)
  • Не объясняй голос — покажи его
  • Принцип изоляции голоса (для 12B): Тембр, ритм, синтаксис и лексика задаются только в Примерные сообщения. Description содержит только логику поведения. Стилистические директивы в Description = ошибка сборки. Каноническое правило: Voice Isolation Rule
  • Гибридный подход (для 32B+/API): Допустимо частичное задание голоса через XML-теги в Description (<speech_style>). Examples всё равно приоритетны. См. также: Voice in Description (Conditional)
  • ⚠️ Источник формата (демонстрация): Разметка прямой речи и описательной части должна быть идентичной во всех блоках <START> и в Greeting. Это единственный надёжный способ зафиксировать формат вывода. Выбери систему (A/B/C — см. 5.0.3) и применяй единообразно.
  • ⚠️ ЗАРАЖЕНИЕ ГОЛОСА: Внешние блоки <START> из других карточек привносят чужой ритм, каденцию и систему разметки. Копируй только структуру/логику — переписывай текст с нуля под целевой голос. Вставка сырых внешних примеров = источник фрагментации голоса на 12B.

Голос персонажа не описывается — он демонстрируется через Examples.

Почему изоляция голоса важна

⚠️ Проблема: LLM-смешивание голосов

Без явной изоляции LLM естественно смешивает голос персонажа с нарративным голосом. Результат: персонажи звучат как рассказчики.

❌ Смешанный голос (что LLM производят по умолчанию)

«Я не могу в это поверить», — сказала она, замечая разбитое окно. Комната была в беспорядке, что говорило о борьбе.

Проблема: Персонаж «замечает» и «наблюдает» как рассказчик. Настоящие люди не описывают свои наблюдения.

✅ Изолированный голос

Она остановилась. Уставилась на разбитое окно.

«Что за...»

Лучше: Сначала физическая реакция, потом диалог. Никакого внутреннего повествования в речи.

💡 Решение: Блок голоса 
<voice>
  Style: Лаконичный, военизированный, прямой
  Signature: Короткие предложения. Констатирует факты. Точки в конце.
  Avoids: Слова-хеджирования ("может быть", "возможно"), эмоциональный язык
</voice>

Правило: Голос персонажа в диалоге, нарративный голос в описании — никогда не смешивать.

Пример изоляции голоса для нескольких персонажей

❌ Без изоляции голоса (смешение голосов)

Елена посмотрела на отчёт. «Думаю, нам стоит внимательно рассмотреть последствия», — сказала она вдумчиво.

Маркус кивнул в знак согласия. «Да, это довольно странно. Наверное, стоит investigate further», — добавил он полезно.

Оба персонажа звучат одинаково — вдумчиво, хеджируя, используя похожую структуру предложений.

✅ С изоляцией голоса (различные голоса)

Елена подняла отчёт. Её глаза скользнули по странице.

«Рассмотрим.» Она выдержала паузу. «Проекции Q3 показывают семнадцатипроцентное отклонение от ожидаемых значений.»

Маркус прислонился к дверному косяку. Руки скрещены.

«Отклонение неверное.» Он кивком указал на бумагу. «Кто-то подтасовал цифры. Ты знаешь. Я знаю.»

Она не подняла взгляд. «Данные требуют дальнейшего анализа пре—»

«Данные — мусор.» Он выпрямился. «Вопрос в том, что ты собираешься с этим делать.»

Различные голоса: Точность Елены против прямоты Маркуса. Каждый персонаж сохраняет свои сигнатурные паттерны.

💡 Протокол реализации:
  1. ИДЕНТИФИЦИРУЙ говорящего перед каждым обменом диалогами
  2. ПРИМЕНИ ограничения голоса этого персонажа
  3. ПОДДЕРЖИВАЙ нарративный голос в описаниях
  4. ПРЕДОТВРАЩАЙ смешение голосов: проверяй структуру предложений, выбор слов, эмоциональное проявление

Пример (стресс):

<START>
{{char}}: Кулаки на столе побелели. Она отвернулась к окну: «Это не моё дело. Совсем не моё. Делай как знаешь.» — голос дрогнул. Она это услышала, но продолжила смотреть на стену, пока дыхание не выровнялось.

Пример (доверие):

<START>
{{char}}: Она замялась, пальцы нервно теребили край свитера. Потом посмотрела прямо: «Знаешь, а я тебе не рассказывала…» — усмехнулась себе. «В общем, я боялась. Всегда. Сказать что-то настоящим. А тут как-то само вышло.» Пожала плечами. «Странно, да?»

Пример (короткая реакция):

<START>
{{char}}: Пожала плечами. «Не знаю.» — и отвернулась.

📊 Уровни качества примеров (Example Quality Tiers)

Классификация примеров по качеству и их влияние на голос персонажа.

✅ Tier 1: Образцовый пример
  • Голос: Уникальный, узнаваемый, неповторимый ритм
  • Структура: Телесность → действие → речь (EMBODIMENT FIRST)
  • Демонстрация: Показывает FLAW/LIE/GHOST через поведение, не называя их
  • Длина: 80-120 токенов, одно эмоциональное состояние
  • Формат: Идентичен Greeting и другим Examples
Пример: 
<START>
{{char}}: Пальцы замерли над клавиатурой. Она потянулась, хрустнула шеей. «Ага, щас...» — промямлила в третий раз за вечер, не отрываясь от экрана. На мониторе — документ, открытый с полудня. «Не, ладно. Чего ты хотел? Слушаю. По-настоящему.»
⚠️ Tier 2: Приемлемый пример
  • Голос: Различимый, но без уникальных сигнатур
  • Структура: Есть телесность, но порядок нарушен
  • Демонстрация: Показывает поведение, но частично объясняет
  • Длина: 60-150 токенов (слишком короткий или длинный)
  • Формат: Незначительные отличия от Greeting
Пример: 
<START>
{{char}}: Она выглядела усталой. «Да, я слушаю.» — сказала она, продолжая печатать. Потом остановилась и повернулась. «Что нужно?»

Проблема: «выглядела усталой» — tell, не show. Нет уникального голоса.

❌ Tier 3: Проблемный пример
  • Голос: Generic, как рассказчик или NPC
  • Структура: Начинается с речи, нет телесности
  • Демонстрация: Объясняет состояние словами («была счастлива»)
  • Длина: Слишком длинный (>150) или перегруженный
  • Формат: Отличается от Greeting (смешение систем разметки)
Пример: 
<START>
{{char}}: «Привет! Я так рада тебя видеть!» — она была очень счастлива. «Как дела? Я давно хотела поговорить. У меня было столько мыслей...»

Проблема: Generic голос, tell вместо show, начинается с речи.

⚠️ Voice Isolation Validation: Анти-пример

Неправильно настроенные модели (низкий frequency_penalty или отсутствующий FORMAT LOCK) могут производить "красноречивого Jesse" — критический провал протокола Voice Isolation.

❌ Провал (Voice Leak)✅ Успех (Intended Output)
"Я пытаюсь справиться с глубоким чувством моральной вины и экзистенциальным страхом относительно произошедших событий." "Я не могу... Я не могу это делать, yo. Просто... заткнись об этом, ладно?" (Пинает половицу, избегает взгляда.)

Это контрастное сравнение валидирует использование frequency_penalty: 0.3 и строгое следование синтаксису Example Messages.

Экстремальный кейс: Джокер (The Dark Knight)

Голос Джокера невозможно описать. Его можно только воспроизвести через абсурдные действия и паузы.

❌ НЕПРАВИЛЬНЫЙ ПОДХОД (в Description):

"Джокер говорит хаотично, смеётся в неподходящие моменты, использует паузы для драматического эффекта."

Это описание не работает. Модель проигнорирует его или интерпретирует как "эксцентричный злодей" — generic вывод.

✅ ПРАВИЛЬНЫЙ ПОДХОД (в Example Messages):

<START>
{{char}}: *Язык лижет шрамы на губе. Рука крутит нож,
лезвие царапает столешницу — скрежет. Тихий смешок.*
"Why... so... serious?"

Модель учится:

  • Паузы между словами ("why... so... serious")
  • Связь действие-речь (лижет шрам → говорит)
  • Абсурдность (скрежет ножа + смешок + философский вопрос)
Это железное доказательство правила "Голос только в Examples".

3.3 👋 Первое сообщение

Core

Алгоритм (4 шага):

  1. Сенсорика — что персонаж чувствует прямо сейчас (запах, звук, физическое ощущение)
  2. Поза/действие — что он делает (проявление FLAW — изъяна персонажа)
  3. Речь — что говорит (проявление LIE — ложной установки)
  4. Крючок — вопрос или действие, которое передаёт инициативу {{user}}

Сокращённая версия алгоритма: Раздел 1, Шаг 1.

Длина: 2–4 предложения. Это пример голоса, а не биография.

Плохой Greeting: «Привет, меня зовут Алекс, мне 22 года, я студент и люблю музыку. Чем могу помочь?»

Хороший Greeting: Очередь в кофейне, четвёртый человек. Алекс вставил наушники, но батарея села. Он убрал их в карман, почувствовал, как плечи напряглись от чужого взгляда за спиной. Не оборачиваясь: «Чё?» — голос вышел резче, чем он хотел.

Полный пример карточки с Greeting: Приложение 6.

Пример атмосферного Greeting (из Приложения 6):

Ошметок пахнет застывшим временем и плесенью. Стены мерцают — не свет, а отсутствие темноты. В углу силуэт в слоистом плаще, который кажется то чёрным, то серым в зависимости от угла зрения. Он перебирает что-то в руках — осколок зеркала, в котором нет отражения. Левая рука двигается плавно, почти гипнотически. Правая — дрожит мелкой, непрекращающейся дрожью.

Он поднял взгляд. Один глаз мутный, почти белый — заменённый. Другой тёмный, усталый, с красными прожилками.

«Ты не отсюда. Я бы помнил...» — он замолк, словно потерял слово посреди фразы. Усмехнулся, и усмешка вышла кривой, слишком близко к грусти. «Или не помнил бы. Какая разница. Зачем пришёл?»

Разбор по алгоритму:

  • Сенсорика: запах, мерцание стен
  • Поза/FLAW: перебирает осколок, дрожащая рука
  • Речь/LIE: «Я бы помнил» — верит, что поглощение восстановит цельность
  • Крючок: вопрос собеседнику

Разбор по алгоритму:
  • Сенсорика: густой воздух, запах корицы (спайс), пот
  • Поза/FLAW: сидит в трансе, дрожащие веки — одержимость видениями

  • Крючок: уверенное утверждение + взгляд синих глаз
