📖 Живой персонаж для LLM __VERSION__ — Гайд по созданию RP-карточек

Базовая модель: 12B+ (Qwen 2.5 14B, Gemma 3 12B, Llama 4 Scout 17B) | Принцип: самодостаточность, поведенческий движок, никаких внешних ссылок.

Что это такое

Engineering pipeline для создания детерминированных RP character cards для LLM (12B–32B+). Карточка персонажа — это поведенческий движок, а не описание.

Для кого

Intermediate prompt engineers, знакомые с базовыми концепциями LLM.

Что вы узнаете

  • Строить карточки персонажей как поведенческие движки
  • Применять психологические фреймворки (OCEAN, Enneagram)
  • Структурировать промпты для стабильного поведения персонажа

Как читать

Линейная прогрессия: Stage 0 (Foundation) → Stage 1 (Quick Start) → ... → Stage 7 (Debugging). Каждый этап строится на предыдущем.

Оценка времени

Полный гайд: ~2–3 часа. Quick Start: ~30 минут.

🔧 Фильтр по модели

Фильтрует контент по размеру модели. Рекомендации для 12B помечены [12B], для 32B+ — [32B+].
