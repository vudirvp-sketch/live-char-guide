📖 Живой персонаж для LLM __VERSION__ — Гайд по созданию RP-карточек

Базовая модель: 12B+ (Qwen 2.5 14B, Gemma 3 12B, Llama 4 Scout 17B) | Принцип: самодостаточность, поведенческий движок, никаких внешних ссылок.

Что это такое

Engineering pipeline для создания детерминированных RP character cards для LLM (12B–32B+). Карточка персонажа — это поведенческий движок, а не описание.

Для кого

Intermediate prompt engineers, знакомые с базовыми концепциями LLM.

Что вы узнаете

  • Строить карточки персонажей как поведенческие движки
  • Применять психологические фреймворки (OCEAN, Enneagram)
  • Структурировать промпты для стабильного поведения персонажа

Как читать

Линейная прогрессия: Stage 0 (Foundation) → Stage 1 (Quick Start) → ... → Stage 7 (Debugging). Каждый этап строится на предыдущем.

Оценка времени

  • Track A (Начинающий): ~15 мин — минимальный шаблон, базовые концепции
  • Track B (Практик): ~30 мин — полный пайплайн, все core-элементы
  • Track C (Продвинутый): ~60 мин — расширенная теория, все опциональные элементы

Полный гайд: ~2–3 часа (все треки последовательно).

🔧 Фильтр по модели

Фильтрует контент по размеру модели. Рекомендации для 12B помечены [12B], для 32B+ — [32B+].

🔗 Карта зависимостей блоков

Кликните на узел для навигации. Стрелки показывают зависимости: A → B означает, что B требует A.

📋 Краткая справка зависимостей

  • Spine → Anchors: WANT/NEED/FLAW/LIE/GHOST определяют триггеры и действия
  • Enneagram → Anchors: Тип определяет страхи и мотивации для якорей
  • OCEAN → Anchors: Экстремальные полюса определяют поведенческие паттерны
  • Anchors → Examples: Якоря демонстрируются в примерах диалогов
  • Anchors → System Prompt: Глобальные правила поведения
  • Spine (GHOST) → Lorebook: Формирующие события хранятся в Lorebook
