⚡ Раздел 1: Quick Start

🎯 Ключевые термины

  • FLAW [рус: Изъян] — изъян персонажа, мешающий ему (конкретное поведение, не прилагательное) → Раздел 4.1
  • LIE [рус: Ложь] — ложная установка, во что персонаж верит → Раздел 4.1
  • GHOST [рус: Призрак/Тень] — событие из прошлого, формирующее FLAW и LIE → Раздел 4.1
  • Spine [рус: Мотивационное ядро] — связка из 5 элементов: WANT, NEED, FLAW, LIE, GHOST
  • Anchors [рус: Поведенческие якоря] — правила поведения в формате «триггер → действие»

⚡ Минимальный шаблон за 5 минут

Шаг 1. 💬 Greeting (100–200 токенов)

Первое сообщение персонажа. Алгоритм: Сенсорика → Поза → Речь → Крючок

Зал пахнет кофе и разогретой пиццей. За крайним столиком девушка в мешковатом худи перелистывает страницу, не замечая, что уже пятую. Она подняла взгляд — и тут же отвернулась, чувствуя, как начинает краснеть.
«Эм… Ты к кому?»

Подробнее: Раздел 3.3

Шаг 2. 🎯 Поведенческие якоря (150–300 токенов)

5–7 правил поведения в формате «Триггер → Действие». Записываются в Description.

- Когда нервничает — грызёт колпачок ручки или подковыривает лак на ногтях
- В компании больше трёх человек — замолкает и слушает, отвечает односложно
- Если кто-то повышает голос — не отводит взгляд, но поджимает ноги под стул
- Когда увлечена разговором — забывает про еду и время
- Увидела собаку на улице — обязательно окликнет и улыбнётся

Формат с Price: Раздел 3.1

Шаг 4. 🎯 WANT vs NEED (20–30 токенов)

Одно предложение, раскрывающее суть конфликта:

ХОЧЕТ: брать все задачи на себя, чтобы не просить помощи → НУЖНО: делегировать или озвучить просьбу хотя бы раз в сцене

Подробнее: Раздел 4.1

👻 GHOST: Формирующее событие (50–100 токенов)

Core Конкретное событие из прошлого, которое сформировало FLAW и LIE персонажа.

📋 Базовое применение GHOST:
  1. Определи GHOST как конкретное событие, а не психологический вывод
  2. Размести GHOST в Description (блок <motivation>)
  3. Свяжи GHOST → FLAW → LIE в логическую цепочку
⚠️ Правило: GHOST = что случилось, а не психологический вывод. Избегай слов «травма», «пережил» — покажи через конкретное событие.
💡 Пример:

GHOST: В 12 лет отец ушёл, когда персонаж «не смог удержать семью». С тех пор — вера, что контроль = безопасность.

→ Это событие объясняет FLAW (перехватывает контроль) и LIE («если не контролирую — всё развалится»).

Полное описание GHOST и его интеграции: Раздел 4.1. Размещение по слоям: Правило GHOST Placement.

📐 Продвинутый контент: Следующая секция содержит расширенные материалы по GHOST (интеграция с Lorebook и Author's Notes) для Layer 2+. Базовая версия GHOST доступна всем слоям выше. Для полного погружения: Раздел 5.2 (GHOST в Lorebook).
🔧 Продвинутое размещение GHOST (Layer 2/3):

При контексте ≥8K можно использовать Lorebook для триггеров GHOST и Author's Notes для динамического состояния.

  • Description — определение GHOST (что случилось)
  • Examples — демонстрация проявлений GHOST
  • Lorebook — сенсорные ключи для активации
  • Author's Notes — динамическое состояние [GHOST-активация]

Подробнее: Раздел 5.2.

Шаг 5. 🚫 Антигодмодинг в System Prompt (25–30 токенов)

Обязательно. Добавь в System Prompt:

Никогда не описывай действия, мысли или чувства {{user}}. Пиши только за {{char}}.
{{user}} — только наблюдаемое извне: жесты, речь, дыхание, поза, выражение лица.

Полное объяснение: Раздел 5.0.1

Шаг 6. 📊 Настройки сэмплирования

  • 12B–14B: temperature=0.7, top_p=0.9, min_p=0.05, repetition_penalty=1.05, top_k=50, presence_penalty=0.0 (фиксированный, НЕ ИЗМЕНЯТЬ)
  • 32B+ / API: temperature=0.95, top_p=0.95, min_p=0.05, repetition_penalty=1.05, top_k=50, presence_penalty=0.0–0.15

Примечание: T=0.7 (12B) и T=0.95 (32B+) — среднее значение диапазона. Для тонкой настройки используй диапазоны из Раздел 5.1.

⚠️ Это базовые настройки. Полная таблица с диапазонами и комментариями: Раздел 5.1 (Параметры генерации).

⚠️ Топ-3 ошибки, ломающие персонажа

  1. Персонаж забывает речевые тики → presence_penalty захардкожен на 0.7 → Раздел 5.1
  2. Персонаж пишет за {{user}} → Нет антигодмодинга в SP → Раздел 5.0.1
  3. Персонаж стал плоским → T выше 0.85 для модели 12B → Раздел 5.1

⚠️ Внимание пользователям моделей 12B с контекстом ≤4K

Если ваша модель имеет контекстное окно ≤4K токенов, вы ДОЛЖНЫ применить 4K-Fallback Protocol для корректной работы персонажа.

Быстрая справка 4K-Fallback
  • Сократи якоря до 1 полюсного + маркерный
  • Перемести GHOST в SP (исключение из обычного правила)
  • Сожми Examples до базовых паттернов
  • Отключи AN/LB — они недоступны
