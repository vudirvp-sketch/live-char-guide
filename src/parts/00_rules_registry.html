📌 Раздел 0.1: Канонический реестр правил

Core — авторитетные определения правил, на которые ссылаются все разделы гайда.

§R1. Price: Обязательность цены

ОПРЕДЕЛЕНИЕ: Price — немедленная физическая или вербальная манифестация триггера, разрешающаяся в той же сцене.

⚠️ КРИТИЧЕСКИ: Будущие состояния («позже будет жалеть», «навсегда запомнит») — НЕ валидный Price. 
Формат: [Trigger] → [Action] → [Price: <immediate manifestation>]

Подробнее: Раздел 0 §2

§R2. Voice Isolation: Изоляция голоса

ОПРЕДЕЛЕНИЕ: Голос персонажа задаётся исключительно в Примерных сообщениях (Examples) для моделей 12B–14B, и преимущественно там для 32B+/API.

§R3. GHOST Placement: Размещение по слоям

ОПРЕДЕЛЕНИЕ: GHOST размещается в Description (определение) + LB/AN (триггеры). System Prompt — только для 4K-Fallback.

Подробнее: Раздел 0 §4

§R4. Anti-Godmodding: Антигодмодинг

ОПРЕДЕЛЕНИЕ: Обязательные строки в System Prompt против управления {{user}}.

Никогда не описывай действия, мысли или чувства {{user}}. Пиши только за {{char}}.
{{user}} — только наблюдаемое извне: жесты, речь, дыхание, поза, выражение лица.

Подробнее: Раздел 0 §6

§R5. Voice in Description (Conditional)

ОПРЕДЕЛЕНИЕ: Голос в Description разрешён ТОЛЬКО для моделей 32B+/API через тег <speech_style>. Для 12B–14B — запрещён.

⚠️ УСЛОВИЕ: Для 12B–14B любые стилистические директивы в Description конфликтуют с примерами или игнорируются моделью.

Подробнее: Раздел 0 §3

§R6. Anchor Format: Формат якорей по стадиям

ОПРЕДЕЛЕНИЕ: Формат якорей зависит от стадии разработки.

📋 Прототипный формат (Stages 1–4)

[Trigger] → [Action] → [Price: TBD]

Позволяет быструю итерацию. Цена помечается как TBD, не опускается.

✅ Канонический формат (Stages 5–7)

[Trigger] → [Action] → [Price: <immediate manifestation>]

Обязательно для production. Все TBD должны быть разрешены.
