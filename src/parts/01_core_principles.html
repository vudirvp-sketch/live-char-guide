§0.2 Критические анти-паттерны (Все треки)

Краткий список критических ошибок. Полный анализ с примерами: Критические анти-паттерны.

ИЗБЕГАЙТЕ ЭТИХ ОШИБОК НА ЛЮБОМ ТРЕКЕ:

  • Раздувание токенов: Описание превышает 800 токенов → вызывает переполнение контекста. Держите под 600.
  • Непроявление Price: Цена указана, но никогда не триггерится в диалоге → ломает логику психологических якорей. → Правило цены
  • Голос в Description: Речь от первого лица в поле Description → конфликтует с голосом SP. Используйте только третье лицо. → Изоляция голоса
  • GHOST в System Prompt: Формирующее событие помещено в System Prompt → должно быть в отдельном поле GHOST. → Размещение GHOST
  • RepPen > 1.10: Repetition Penalty выше 1.10 → вызывает несвязный вывод. Рекомендуется: 1.0–1.05.
  • Godmoding в LB: Записи LB, управляющие действиями пользователя → нарушает согласие RP. Только автономия игрока. → Антигодмодинг
⚠️ Track A Users: Полный каталог анти-паттернов с примерами требует Track C. Для базовых исправлений используйте Диагностику (Раздел 7).

§1. OCEAN: Правило 1–2 полюсов

— В карточку идёт только 1–2 экстремальных полюса OCEAN (значения <30 или >70)
  • ✓ 1–2 экстремума — персонаж запоминающийся, поведение последовательное
  • ⚠ 0 экстремумов — персонаж может быть забываемым, «усреднённым»
  • ✗ 3+ экстремума — персонаж может быть непоследовательным, конфликтующим
⚠️ Полный OCEAN + 7 якорей → перегруз контекста.

Модель теряет фокус, голос «плывёт». Приоритет резания: удаляй всё, что не генерирует наблюдаемое действие, реакцию или цену.

Полное руководство по OCEAN: Раздел 4.3. Справочные таблицы: Приложение 2.

§2. Price: Обязательность цены

— Price — обязательный элемент для критичных якорей, рекомендуемый для остальных

КАНОНИЧЕСКИЙ ФОРМАТ ЯКОРЕЙ:

[Trigger] → [Action] → [Price: <immediate physical/verbal manifestation>]
(рус: [Внешний стимул → Наблюдаемая реакция → Мгновенное физическое или речевое последствие в текущей сцене])
⚠️ ПРАВИЛО: Цена должна разрешаться в той же сцене.

Price — это немедленная физическая или вербальная манифестация, а не отложенное последствие. Будущие состояния («позже будет жалеть», «навсегда запомнит») — НЕ валидный Price.

Пример: Уолтер Уайт (Breaking Bad) — Price как доказательство

[Trigger] Скайлер спрашивает про второй телефон

[Action] Уолтер касается виска, поправляет очки,
         говорит медленно: "Какой телефон?"

[Price] Пауза тянется на две секунды дольше нормы.
        Зрачки сужаются, большой палец трёт оправу —
        стереотипная микроэкспрессия лжеца.
        Скайлер это видит. Иллюзия "Гейзенберга" рушится физически.
Почему этот пример важен:

Price — это не всегда "плач", "крик" или "обморок". Это может быть микроэкспрессия, выдающая ложь. Пауза. Взгляд. Изменение темпа речи.

Price = визуальное доказательство для читателя/собеседника в RP.
Персонаж становится уязвимым, даже если он "сильный".

Цена обязательна для якорей, связанных с:

  • FLAW — изъян персонажа должен иметь наблюдаемые последствия
  • GHOST — триггеры прошлого должны манифестироваться физически

Полный протокол якорей: Раздел 3.1.

📋 Условное размещение GHOST

УсловиеРазмещениеПричина
По умолчаниюSpine (Description) + LB/ANСохраняет SP для основных инструкций

Уникальный кейс: Безымянный (Planescape: Torment)

В Planescape: Torment GHOST — буквально двигатель сюжета, забвение Практичного Воплощения. Это единственная вселенная, где механика GHOST = игровая механика.

[Trigger] Упоминание имени "Деонарра" или запах бальзамирующей жидкости

[Action] Безымянный замирает. Кожа на предплечье вздувается рунами боли.
        Глаза смотрят сквозь собеседника.

[Price] Диалог обрывается фразой:
        "Я не знаю этого человека... Но мои шрамы знают."
        Персонаж физически не может продолжить разговор.
        Руки дрожат так сильно, что приходится сесть.
Почему этот пример важен:

Это демонстрирует, что GHOST — не просто "персонаж грустит о прошлом". GHOST = механика, которая контролирует поведение при встрече с триггером.

Персонаж теряет контроль не потому что "ему грустно", а потому что так работает проклятие Бессмертного.

В контексте RP: GHOST-якорь с валидным Price = момент, когда игрок видит травму, а не узнаёт о ней из монолога.

⚠️ 4K-Fallback — редкое исключение: Всегда предпочитайте Spine + LB/AN для GHOST-токенов. GHOST в SP — только при экстремальном лимите контекста (~300 токенов). См. Раздел 5.12 (4K-Fallback).

Карта слоёв GHOST: Раздел 5.2.
