🔧 Диагностика для моделей 12B–14B

ℹ️ Контекст: Модели 12B–14B (Qwen 2.5 14B, Gemma 3 12B, Llama 4 Scout 17B, Mistral 12B) имеют ограниченный контекст и меньшую способность следовать сложным инструкциям.
СимптомПричинаРешение
Голос «плывёт» Voice Leak в Description Удали стилистические директивы из Description. Голос только в Examples. → §R2
Персонаж OOC T > 0.85 Снизь temperature до 0.6–0.8
Годмодинг Нет антигодмодинга Добавь блок в SP → §R4
Повторяет фразы Token bloat Сократи Description до <600 токенов
Игнорирует якоря Нет Price Добавь Price к критичным якорям → §R1
Забывает контекст Контекст ≤4K Примени 4K-Fallback Protocol
Теряет стиль речи presence_penalty > 0 Установи presence_penalty = 0.0 (зафиксируй!)
«Умный» персонаж Голос в Description Удали любые описания речи. Только демонстрация в Examples.
⚠️ Voice Isolation критичен для 12B: Любой голос в Description конфликтует с Examples. Только демонстрация.

Каноническое правило: §R2. Voice Isolation

🔧 Рекомендуемые настройки для 12B–14B 
temperature = 0.6–0.8      # НЕ выше 0.85
top_p = 0.9
min_p = 0.05
repetition_penalty = 1.05   # НЕ выше 1.10
top_k = 50
presence_penalty = 0.0      # ЗАФИКСИРОВАНО! Не менять

presence_penalty = 0.0 обязателен для 12B. Значения > 0 «убивают» речевые тики и уникальный голос.

Core

Что это: Базовые симптомы и решения для Track B.

⚠️ Обязательно: Прочитайте Критические анти-паттерны перед этим разделом.

🔀 Симптом 1: Персонаж нарушает роль / выходит из образа

— Проверь Spine, якоря, SP и Examples на согласованность
  1. Проверь Spine: Ясно ли определена мотивация?
    • Если неоднозначно → Уточни Spine: конкретизируй WANT/NEED/FLAW/LIE
  2. Проверь якоря: Согласованы ли триггеры с личностью?
    • Если противоречат → Перестрой якоря: выровняй по Spine и OCEAN
  3. Проверь SP: Есть ли явные ограничения?
    • Если нет → Добавь ФИЛЬТР ПЕРЕД ГЕНЕРАЦИЕЙ (для 12B)
  4. Проверь Examples: Демонстрируют ли они нужное поведение?
    • Если противоречат → Перепиши Examples: они приоритетны для голоса

🔀 Симптом 2: Нестабильный голос / голос «плывёт»

— Проверь Examples, Description, SP и разметку на единообразие
  1. Проверь Examples: Есть ли минимум 2 блока <START>?
    • Если нет → Добавь Examples: нейтральный + стресс
  2. Проверь Description: Есть ли стилистические директивы?
    • Если да → Удали: голос должен быть только в Examples (для 12B) → §R2
  3. Проверь SP: Есть ли голосовые директивы?
    • Если да → Перемести в Examples: SP только для ограничений
  4. Проверь разметку: Единообразна ли во всех блоках?
    • Если нет → Унифицируй: выбери одну систему (A/B/C) для всех

🔀 Симптом 3: Галлюцинации / противоречивые факты

— Проверь LB, якоря, OCEAN и GHOST на противоречия
  1. Проверь LB: Факты противоречат друг другу?
    • Если да → Устрани противоречия: оставь один источник истины
  2. Проверь якоря: Есть ли Price для FLAW/GHOST?
    • Если нет → Добавь Price: последствия должны быть определены → §R1
  3. Проверь OCEAN: Больше 2 экстремальных полюсов?
    • Если да → Сократи: оставь только 1–2 экстремума → §4.3
  4. Проверь GHOST: Дублируется ли в нескольких местах?
    • Если да → Устрани дубликаты: GHOST только в Description, триггеры в LB/AN → §R3

🔀 Симптом 4: Годмодинг (модель пишет за {{user}})

— Проверь SP, Examples и LB на наличие действий {{user}}
  1. Проверь SP: Есть ли антигодмодинг?
    • Если нет → Добавь: «Никогда не описывай действия, мысли или чувства {{user}}» → §R4
  2. Проверь Examples: Есть ли действия {{user}} в примерах?
    • Если да → Удали: примеры не должны содержать {{user}} действия
  3. Проверь LB: Есть ли content с действиями {{user}}?
    • Если да → Удали: это нарушает антигодмодинг

🔀 Симптом: Персонаж «слишком идеальный»

— Проверь FLAW, WANT/NEED конфликт и GHOST на поведенческие проявления
  1. Проверь FLAW: Проявляется ли в поведении?
    • Если нет → Добавь якоря с Price: FLAW должен иметь последствия
  2. Проверь WANT/NEED: Есть ли конфликт?
    • Если WANT = NEED → Переработай: NEED должен быть противоположен WANT
  3. Проверь GHOST: Влияет ли на поведение?
    • Если нет → Добавь триггеры: GHOST должен активироваться через сенсорные ключи

🔀 Симптом: Повторы / зацикливание

— Проверь RepPen, presence_penalty и разнообразие триггеров
  1. Проверь RepPen: Значение > 1.10?
    • Если да → Снизь до 1.05: высокий RepPen стирает паттерны
  2. Проверь presence_penalty: Значение > 0.15?
    • Если да → Снизь до 0.0–0.15: высокие значения убивают тики
  3. Проверь якоря: Есть ли разнообразие в триггерах?
    • Если нет → Расширь покрытие: добавь ситуации разного типа

🔀 Симптом: Потеря контекста в длинной сессии

— Проверь AN, размер окна и Lorebook Range-cascade
  1. Проверь AN: Обновляется ли каждые 3–5 сообщений?
    • Если нет → Настрой протокол обновления AN
  2. Проверь контекст: Какой размер окна?
    • Если 4K → Прими ограничения: используй 4K-Fallback протокол
  3. Проверь LB: Есть ли Range-cascade для прогрессии?
    • Если нет → Добавь: каскад по диапазону сообщений

🔀 Симптом: Физические противоречия (телепортация, анатомия)

— Проверь SPATIAL LOCK и Examples на пространственную когерентность
  1. Проверь SP: Есть ли SPATIAL LOCK?
    • Если нет → Добавь: «Track distance, posture, object weight, sightlines. No teleportation.»
  2. Проверь Examples: Демонстрируют ли они пространственную когерентность?
    • Если нет → Добавь примеры: с явным отслеживанием позиции
💡 Правило одного изменения:

Никогда не меняй более одного параметра за раз. Иначе не узнаешь, что помогло. После каждого изменения запускай соответствующий тест-сценарий.
