🎯 Заключение

Что вы узнали

  • Архитектура карточки: Карточка персонажа — это поведенческий движок, а не описание. Каждый элемент должен производить наблюдаемое действие.
  • Spine (мотивационное ядро): 5 элементов — WANT, NEED, FLAW, LIE, GHOST — определяют причину поведения.
  • Психологические профили: Enneagram + OCEAN (только 1–2 экстремальных полюса) создают последовательный характер.
  • Поведенческие якоря: Формат [Триггер] → [Действие] → [Price] превращает психологию в наблюдаемое поведение.
  • Изоляция голоса: Голос задаётся только в Examples, Description — только логика поведения.
  • Технические ограничения: System Prompt = антигодмодинг + SPATIAL LOCK + FORMAT LOCK.
  • Динамика: AN и LB поддерживают живость в длинных сессиях.

Ключевые правила (напоминание)

  1. OCEAN: Только 1–2 экстремальных полюса
  2. Price: Обязателен для FLAW/GHOST якорей
  3. Voice: Только в Examples, никогда в Description (для 12B)
  4. GHOST: Только в Description, никогда в SP
  5. trigger → action → price: Каноническая схема поведения
⚠️ Перед деплоем:

Убедитесь, что все проверки из Приложения 4 пройдены.
