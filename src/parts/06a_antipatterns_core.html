📌 Раздел 6a: Критические анти-паттерны

Warning — эти 7 ошибок критически влияют на качество персонажа на ЛЮБОМ треке.

❌ AP-1: Раздувание токенов

ПРОБЛЕМА: Description превышает 800 токенов → вызывает переполнение контекста, модель теряет фокус.

РЕШЕНИЕ: Держите Description под 600 токенов. Удаляйте всё, что не генерирует наблюдаемое действие, реакцию или цену.

ПРИМЕР:

❌ Плохо: 1200 токенов с детальной биографией, историей мира, описанием одежды
✅ Хорошо: 450 токенов с Spine, 2-3 якорями, кратким GHOST

❌ AP-2: Непроявление Price

ПРОБЛЕМА: Цена указана в якоре, но никогда не триггерится в диалоге → ломает логику психологических якорей.

РЕШЕНИЕ: Price должен быть наблюдаемым физическим или вербальным проявлением в той же сцене.

ПРИМЕР:

❌ Плохо: "Её доверие будет подорвано навсегда" (отложенное, не наблюдаемое)
✅ Хорошо: "Дрожат руки, голос срывается на шёпот" (мгновенное, наблюдаемое)

Кейс-стади: Синдзи Икари (Evangelion) — Синхронизация с Евой-01

❌ АНТИ-ПАТТЕРН (отложенный price):

"Синдзи боялся боя и позже плакал в раздевалке."

Это невалидный Price по двум причинам:

  1. "Позже" — нарушает правило мгновенности
  2. "Плакал" — внутреннее состояние, не наблюдаемое в сцене боя

✅ ПРАВИЛЬНЫЙ PRICE (мгновенная манифестация):

"В момент удара Ангела Синдзи *кричит одновременно с Евой*,
сгибается пополам в кокпите, из носа течёт кровь синхронизации.
Боль пилота = урон робота. Price неразделим с действием."
Почему Evangelion — идеальная метафора:

В этой вселенной боль пилота всегда совпадает с уроном Евы. Это физический закон мира, а не просто стилистический выбор.

Для новичков это самый наглядный пример:
Нельзя написать "он расстроится позже".
Price — СЕЙЧАС ИЛИ НИКОГДА.

❌ AP-3: Голос в Description

ПРОБЛЕМА: Речь от первого лица или стилистические директивы в Description для 12B–14B → конфликтует с голосом из SP/Examples.

РЕШЕНИЕ: Description — только третье лицо, факты. Голос задаётся в Examples.

ПРИМЕР:

❌ Плохо: "Я говорю с сарказмом и использую слово 'чувак'"
✅ Хорошо: "Елена — саркастичный журналист, использует неформальный тон"

❌ AP-4: GHOST в System Prompt

ПРОБЛЕМА: Формирующее событие помещено в System Prompt вместо отдельного поля → переполнение, потеря приоритета.

РЕШЕНИЕ: GHOST размещается в Description (определение) + LB/AN (триггеры). SP — только для 4K-Fallback.

ИСКЛЮЧЕНИЕ: 4K-Fallback — при context_window ≤ 4K и недоступности User Context, можно одно предложение в SP.

❌ AP-5: RepPen > 1.10

ПРОБЛЕМА: Repetition Penalty выше 1.10 → вызывает несвязный вывод, потерю стиля.

РЕШЕНИЕ: Диапазон 1.0–1.05. Для решения проблем с повторами лучше добавить вариативность в Examples.

РЕКОМЕНДАЦИЯ:

✅ 1.0–1.05 — оптимальный диапазон
⚠️ 1.05–1.10 — допустимо при проблемах с повторами
❌ > 1.10 — вызывает деградацию качества

❌ AP-6: Godmoding в Lorebook

ПРОБЛЕМА: Записи LB, управляющие действиями пользователя → нарушает согласие RP, ломает автономию игрока.

РЕШЕНИЕ: LB описывает только реакцию персонажа, не действия {{user}}.

ПРИМЕР:

❌ Плохо: "{{user}} чувствует страх и убегает"
✅ Хорошо: "При приближении {{user}} отступает на шаг, сжимает оружие"

❌ AP-7: OCEAN Overload

ПРОБЛЕМА: 3+ экстремальных полюса OCEAN → персонаж непоследовательный, конфликтующий.

РЕШЕНИЕ: Только 1–2 экстремальных полюса (значения <30 или >70). 0 экстремумов = забываемый персонаж.

ПРИМЕР:

✅ 1–2 экстремума — запоминающийся, последовательный
⚠️ 0 экстремумов — может быть "усреднённым"
❌ 3+ экстремума — непоследовательный
