🏗 Раздел 2: Архитектура системы

📋 SCOPE: Обязательная полная схема (5-7 якорей, Price, GHOST).
Этот гайд — инженерный пайплайн, а не набор советов. Система строится от внутреннего ядра к внешнему исполнению.

Сдвиг парадигмы: от описания к поведенческому движку

Карточка трансформируется из статичного описания в детерминированный поведенческий движок. Художественные директивы («показывай, не рассказывай») заменяются механическими триггерами. Модель не интерпретирует контекст — она исполняет скомпилированную инструкцию: Триггер → Действие → Цена. Это гарантирует тактильность, сюжетный вес и стабильность голоса на архитектурах 12B–32B+ без раздувания контекста.

Ключевой принцип: Каждый элемент карточки должен активно влиять на генерацию, а не пассивно описывать персонажа. Если блок не производит наблюдаемое действие — он должен быть удалён или переработан.

6 процессуальных правил (ядро системы)

Core — эти правила — фундамент поведенческого движка. Они применяются ко всем элементам карточки.

#ПравилоСуть
1Конвертер «Статика → Якорь»Все детали внешности/лора проходят фильтр: [Условие → Наблюдаемое действие → Физиологическая/психологическая цена]. Абстрактные списки (без связи с действием) не работают — модель их игнорирует.
2Телесность (Core)Внутреннее состояние манифестируется через вес, баланс, дыхание или микродвижения до речи. Шаблон: [Среда/состояние] → [реакция тела] → [сенсорная обратная связь].
3Протокол 4K-FallbackПри лимите ~300 токенов: Greeting (сенсорика+локация+тон) + 3 Поведенческие якоря (привязаны к физике мира). Лорбук отключён. Сеттинг существует только в форме действия/реакции. Полный протокол — Раздел 5.12.
4GHOSTВизуальные/тактильные маркеры = триггеры памяти. Дублировать как Keywords в Лорбук (при наличии контекста) или вшивать в якоря. Путь модели: триггер → реакция тела → скрытый паттерн.
5Динамическая арка & ГраницыФиксированные вехи заменены на WANT/NEED. AN (Action Note) обновляется каждые 3–5 сообщений: фокус смещается с WANT к поведению, зеркалящему NEED. Влияние на {{user}} только через наблюдаемые симптомы → реакция {{char}}. Годмодинг устранён правилом: {{user}} — только наблюдаемое извне.
6Изоляция голосаТембр, ритм, синтаксис и лексика задаются исключительно в Примерных сообщениях. Description содержит только логику поведения. Любые стилистические директивы в Description = ошибка сборки. Исключение: <speech_style> в System Prompt разрешён только для 32B+/API. Никогда не помещайте стилистические директивы в Description.

Пайплайн создания (10 шагов)

  1. Spine → WANT/NEED/FLAW/LIE/GHOST (через физиологию)
  2. Enneagram → Тип + крыло + стресс→рост (1–2 строки)
  3. OCEAN → Только 1–2 экстремальных полюса
  4. Anchors → 5–7 правил Триггер → Действие
  5. Первое сообщение → Сенсорика → Поза(FLAW) → Речь(LIE) → Крючок
  6. Examples → 3 блока: нейтральный / стресс / доверие
  7. System Prompt → Антигодмодинг + стиль + XML-теги
  8. Параметры генерации → T, Top P, Min P, RepPen=1.05, presence_penalty=0.0–0.15
  9. Динамика → AN (обновлять каждые 3–5 msg при 8K+) + Лорбук
  10. Проверка → 4 тест-сценария → таблица симптомов [→ Приложение 4]

Схема единого потока

[Психология] Spine → Enneagram → OCEAN (1-2 полюса)
      ↓
[Процессуальные правила] Статика→Якорь → Телесность → 4K-Fallback → GHOST → Динамическая арка → Изоляция голоса
      ↓
[Поведение] Anchors (5-7, с ценой) → Greeting → Examples (3 типа)
      ↓
[Ограничения] SP: антигодмодинг + SPATIAL LOCK + PRE-GENERATION FILTER + XML-теги
      ↓
[Запуск] Sampling: T/TopP/MinP/RepPen=1.05 + presence_penalty=0.0–0.15
      ↓
[Поддержка] AN (обновление 3-5 msg, WANT→NEED) + Lorebook (GHOST/мир/NPC)
      ↓
[Отладка] 4 тест-сценария → симптомы → резание → стабильность

Итог: Система функционально замкнута и внутренне непротиворечива. Карточка — поведенческий движок, а не описание. Рабочий минимум: Spine + Anchors (с ценой) + 3 Examples + SP (с Предгенерационный фильтр). При 4K — использовать Fallback-протокол. Остальное наращивается по мере контекста.
