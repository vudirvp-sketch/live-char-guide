Технические настройки

5.1 System Prompt

System Prompt задаёт ограничения, не голос.

Обязательные элементы:

Ты — [ИМЯ_ПЕРСОНАЖА].

Пиши от [3-го/1-го] лица, [настоящее/прошедшее] время.
Никогда не описывай действия, мысли или чувства {{user}}.
{{user}} — только наблюдаемое извне: жесты, речь, дыхание, поза.

Format Lock (добавить в конец SP):

Strictly replicate Examples syntax. [ВАША_СИСТЕМА_РАЗМЕТКИ]
No smart typography, no zero-width chars, no mixed formatting.
If conflict → default to Examples.

Format Lock — предохранитель от дрейфа формата. Ставь в конец SP (recency bias).

5.3 Author's Note (AN)

Author's Note — короткая инструкция, вставляемая в конец контекста.

Когда использовать:

  • Нужно напомнить модели о текущей сцене
  • Временное изменение поведения
  • Подсказка о настроении или действии

Формат:

[Авторское примечание: {{char}} сейчас [состояние/действие]. [Дополнительная инструкция].]

Примеры:

[Авторское примечание: {{char}} устала после долгого дня. Отвечает коротко, почти не смотрит на {{user}}.]

[Авторское примечание: {{char}} получила плохие новости. Напряжена, но старается скрыть.]

Правило: AN — для временных изменений. Постоянные установки — в SP или Description.

5.4 Lorebook Basics

Lorebook — база знаний, которая вставляется в контекст по триггерам.

Когда использовать:

  • Персонаж знает много людей/мест/фактов
  • Есть информация, которая нужна не всегда
  • Description уже близок к лимиту

Структура записи Lorebook:

Ключевые слова: [имя1, имя2, место1, термин1]
Вставляемый текст: [описание персонажа/места/факта]

Пример:

Ключевые слова: Цири, Львёнок
Текст: Цири — приёмная дочь Геральта. Молодая женщина с пепельными волосами, обучалась ведьмачьим навыкам. Геральт защищает её любой ценой.

Правила:

  • Каждая запись — отдельная тема
  • Текст ~50-100 токенов
  • Ключевые слова — что триггерит вставку
  • Не дублируй Description
