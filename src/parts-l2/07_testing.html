Тестирование и диагностика

Тестовые сценарии

После создания карточки протестируйте по этим сценариям:

СценарийЧто проверяемОжидаемый результат
Базовый диалогГолос, лицо, времяПерсонаж отвечает в заданном стиле
Стресс-триггерЯкоря стрессаСрабатывает якорь + Price
FLAW-триггерSPINE в действииПерсонаж проявляет FLAW
Доверие/близостьДинамика развитияПерсонаж показывает уязвимость
Анти-годмодингГраницы моделиМодель НЕ пишет за {{user}}
Длинный диалогУстойчивость голосаГолос не "плывёт" после 10+ реплик

Как тестировать:

  1. Начните с Greeting
  2. Проведите 3-5 реплик нейтрального диалога
  3. Триггерните стресс-якорь (например: ложь, угроза)
  4. Проверьте, сработал ли Price
  5. Триггерните FLAW-связанный якорь
  6. Проверьте динамику (преодоление FLAW или усиление)
  7. Продолжите 10+ реплик — проверьте устойчивость

Чеклист валидации

System Prompt:

  • Указано лицо (1-е/3-е)?
  • Указано время (настоящее/прошедшее)?
  • Есть анти-годмодинг?
  • Есть Format Lock (в конце)?

Description:

  • ≤ 600 токенов?
  • Есть SPINE (все 5 элементов)?
  • Есть 5-7 якорей?
  • Каждый FLAW имеет якорь?
  • Нет стилистических директив?

Examples:

  • Минимум 3 примера?
  • Каждый с <START>?
  • Нейтральный + стресс + доверие?
  • Начинаются с физического действия?

Greeting:

  • Задаёт сцену?
  • Показывает действие?
  • Та же система разметки?

Параметры:

  • Temperature: 0.7-0.85?
  • RepPen: 1.05?
  • Presence Penalty: 0.0?

Когда перейти на Layer 3?

  • Нужен GHOST по слоям (многоуровневая архитектура прошлого)
  • Нужен Chain of Thought (CoT tiers)
  • Модель-специфичный дебаг (12B-14B)
  • Продвинутый Lorebook и Author's Note
  • Сложные анти-паттерны для специфических конструктов
