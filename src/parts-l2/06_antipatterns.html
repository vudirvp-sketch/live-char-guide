Критические анти-паттерны

Warning — эти 8 ошибок критически влияют на качество персонажа.

❌ AP-1: Раздувание токенов

ПРОБЛЕМА: Description превышает 800 токенов → модель теряет фокус, начинает повторяться.

РЕШЕНИЕ: Держите Description под 600 токенов. Удаляйте всё, что не генерирует наблюдаемое действие.

ПРИМЕР:

❌ Плохо: 1200 токенов с детальной биографией, историей мира, описанием одежды
✅ Хорошо: 450 токенов с SPINE, 5 якорями, кратким GHOST

❌ AP-2: Непроявление Price

ПРОБЛЕМА: Цена указана в якоре, но не наблюдаема в сцене → ломает логику поведения.

РЕШЕНИЕ: Price — мгновенная физическая реакция в той же сцене.

ПРИМЕР:

❌ Плохо: "Её доверие будет подорвано навсегда" (отложенное)
✅ Хорошо: "Дрожат руки, голос срывается на шёпот" (мгновенное)

❌ AP-3: Голос в Description

ПРОБЛЕМА: Стилистические директивы в Description конфликтуют с Examples.

РЕШЕНИЕ: Description — только факты и якоря. Голос — только в Examples.

ПРИМЕР:

❌ Description: "Говорит саркастично, использует слово 'чувак'"

✅ Description: "Елена — циничный журналист."

✅ Example:
<START>
Она закатила глаза. "О, чувак, ещё один гений."

❌ AP-4: GHOST в System Prompt

ПРОБЛЕМА: GHOST размещён в SP вместо Description → модель не использует его правильно.

РЕШЕНИЕ: GHOST — часть SPINE, размещается в Description.

ПРИМЕР:

❌ System Prompt: "Ты — персонаж с травмой из-за того, что отец ушёл..."

✅ Description: "GHOST: В 12 лет отец ушёл. Решил — нельзя быть слабым."

❌ AP-5: Repetition Penalty > 1.10

ПРОБЛЕМА: RepPen выше 1.10 "убивает" речевые тики и уникальный голос.

РЕШЕНИЕ: RepPen = 1.05, НЕ выше 1.10.

ПРИМЕР:

❌ RepPen = 1.15 → персонаж теряет уникальные обороты
✅ RepPen = 1.05 → повторы контролируются, голос сохраняется

❌ AP-6: Отсутствие анти-годмодинга

ПРОБЛЕМА: Модель описывает действия {{user}} → ломает иммерсию.

РЕШЕНИЕ: Обязательная строка в System Prompt.

ОБЯЗАТЕЛЬНО:

Никогда не описывай действия, мысли или чувства {{user}}.
{{user}} — только наблюдаемое извне: жесты, речь, дыхание, поза.

❌ AP-7: Presence Penalty > 0

ПРОБЛЕМА: Presence Penalty > 0 штрафует уникальные обороты персонажа.

РЕШЕНИЕ: Presence Penalty = 0.0, зафиксировано.

ПОЧЕМУ:

Presence Penalty штрафует любое повторение токена, даже если это signature персонажа (любимое слово, оборот речи).

❌ presence_penalty = 0.5 → "чувак" исчезает из речи
✅ presence_penalty = 0.0 → голос сохраняется

❌ AP-15: OCEAN Overload

ПРОБЛЕМА: 3+ экстремальных полюса OCEAN (значения <30 или >70) → персонаж противоречив, конфликтующие поведения, модель путается.

РЕШЕНИЕ: Ограничь 1-2 полюсами максимум. Если 3-й полюс кажется необходимым → добавь GHOST обоснование. Каждый полюс должен поддерживать FLAW или объяснять LIE.

ПРИМЕР:

ScenarioResult
High E + Low C + High N Тайлер с тревогой = тревожный хаотичный = generic villain (Тайлер "работает" именно потому, что он NOT anxious — он представляет свободу ОТ тревоги)
High O + High C + Low A Творческий перфекционист-эгоист = внутренний конфликт 
❌ OCEAN: O:75 C:20 E:85 A:35 N:80 → 4 экстремума → персонаж непоследователен
✅ OCEAN: O:45 C:15 E:85 A:40 N:30 → 2 экстремума (Low C, High E) → запоминающийся персонаж
