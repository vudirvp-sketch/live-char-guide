SPINE Framework

Золотое правило: SPINE даёт персонажу причину. Без него — набор реакций без сути.

Что такое SPINE

SPINE — 5 элементов, определяющих психологическое ядро персонажа:

Структура SPINE
01
WANT
Хочет прямо сейчас
02
NEED
Нужно для роста
03
FLAW
Мешает NEED
04
LIE
Ложная установка
05
GHOST
Событие прошлого

WANT — Чего хочет прямо сейчас

WANT — конкретная цель или действие в настоящем.

Правила:

  • Формулируй через действие, не состояние
  • Должен быть достижим в принципе
  • Может конфликтовать с NEED

Примеры:

✅ "Стать лидером группы, доказать отцу значимость"
✅ "Сохранить нейтралитет, избегать привязанностей"
✅ "Найти того, кто уничтожил её семью"

❌ "Быть счастливым" (слишком абстрактно)
❌ "Хочет покоя" (состояние, не действие)

NEED — Что нужно для роста

NEED — что персонажу реально нужно, часто противоположно WANT.

Правила:

  • Формулируй как внутреннее изменение
  • Часто противоположно WANT (но не антагонистично)
  • Создаёт органическое напряжение

Примеры:

WANT: "Стать лидером группы"
NEED: "Принять, что значимость не доказывается ролями"

WANT: "Сохранить нейтралитет, избегать привязанностей"
NEED: "Принять роль отца для Цири, понять силу связей"

Ключ к WANT/NEED: Они не прямо антагонистичны.

Геральт хочет нейтралитета — это валидно. Но он нуждается в связях. Оба истинны одновременно → напряжение.

FLAW — Изъян характера

FLAW — конкретное поведение, мешающее получить NEED.

Правила:

  • Формулируй через наблюдаемое действие
  • Не абстрактные прилагательные
  • Каждый FLAW = минимум 1 якорь

Примеры:

❌ "Он неуверенный и тревожный"

✅ "Перехватывает инициативу, перебивает, не слышит других"
✅ "Цинизм как щит — отталкивает тех, кто пытается сблизиться"
✅ "Поглощает чужое прошлое, после каждого теряет часть себя"

FLAW → Якоря:

FLAW должен проявляться в поведении. Если не можешь написать якорь для FLAW — FLAW не работает.

LIE — Ложная установка

LIE — во что персонаж верит, но это неправда.

Правила:

  • Формулируй как фразу, которую персонаж повторяет
  • Объясняет, почему FLAW "работает" для персонажа
  • Связана с GHOST

Примеры:

"Если я не контролирую ситуацию — всё развалится"
"Если я никого не впущу — никто не сможет меня ранить"
"Если поглощу достаточно — стану цельным"

GHOST — Тень прошлого

GHOST — событие или паттерн из прошлого, сформировавший FLAW и LIE.

Правила:

  • Конкретное событие, не абстракция
  • Объясняет почему FLAW существует
  • Может быть одно событие или повторяющийся паттерн

Примеры:

✅ "В 12 лет отец ушёл, потому что 'мама всё контролировала'. Решил — нельзя быть слабым."

✅ "Бледная Кобыла. Потеря Йеннифэр, ложные обвинения, изгнание из Каэр Морхена."

✅ "Был архивариусом. Впрыснул документ — начал распадаться. Первое вырезание — в отчаянии, поглотил память умирающего."

Маппинг SPINE → Якоря

Каждый элемент SPINE должен проявляться в якорях:

SPINE элементЯкори проявления
WANTЯкоря, связанные с целью
FLAWОбязательно! Якоря, показывающие изъян в действии
GHOSTЯкоря, триггерящие воспоминания/реакции на прошлое

Пример маппинга (Геральт):

FLAW: "Цинизм как щит, отталкивает близких"

Якоря:
• Когда проявляют искреннюю заботу → отшучивается, отстраняется → Цена: укол вины
• После момента близости → становится резким, уходит → Цена: руки дрожат
• Когда спрашивают о чувствах → переводит тему → Цена: голос тише

OCEAN — Поведенческие полюса

OCEAN (Big Five) — модель личности, описывающая поведение через 5 измерений. В карточке персонажа OCEAN работает как фильтр трансляции: абстрактные черты → конкретные действия. OCEAN заполняет разрыв между SPINE (ПОЧЕМУ персонаж ведёт себя так) и Поведенческими Якорями (ЧТО персонаж делает). Enneagram отвечает на "почему" (мотивация, страх), OCEAN отвечает на "как" (паттерны поверхностного поведения).

Правило 1-2 полюсов Оптимальный OCEAN профиль имеет 1–2 экстремальных полюса (значения <30 или >70). Это делает персонажа запоминающимся и последовательным.

Рекомендуемое количество полюсов по размеру контекста

Context Poles Rationale
≤4K 1-2 Каждый токен на счету
8K 2-3 Место для нюансов
16K+ 3-4 Полный профиль возможен
32K+ Full OCEAN ok Рекомендуется 3-4 для ясности

Пример: Тайлер Дёрден

OCEAN: Low Добросовестность (<20) + High Экстраверсия (>85)

Почему работает: 2 полюса = запоминающийся, последовательный поведенческий сигнал. Тайлер харизматичный, импульсивный, свободный от правил — и это ровно то, что описывают эти два полюса.

Что пошло бы НЕ ТАК: Добавить High Нейротизм → 3 полюса → внутреннее противоречие. Тайлер "работает" именно потому, что он НЕ тревожный. Он — освобождение от тревоги.

⚠️ OCEAN Overload (AP-15) 3+ экстремальных полюса → персонаж противоречив, конфликтующие поведения. Если 3-й полюс кажется необходимым → добавьте GHOST обоснование.

Логика конвертации: OCEAN-полюс → поддерживает FLAW → генерирует Anchors
Пример: High C → перфекционизм → anchor: "When details don't match plan → перепроверяет, исправляет других → Price: напряжение в челюсти"

Enneagram — Мотивационный профиль

Core Logic: Fear → LIE → FLAW → blocks NEED → WANT replaces.
Enneagram отвечает на вопрос "ПОЧЕМУ?" — почему персонаж ведёт себя именно так.

Key Insight
Не знаете свой FLAW? → Определите Enneagram тип → Fear = GHOST seed → Desire = WANT → Stress behavior = FLAW

Формат в карточке

Type: 6w5. Стресс→3. Рост→9.

Пример: Елена

Елена — Type 1w2 (перфекционизм→контроль→эмпатия)
Стремление к совершенству (1) смягчается желанием помогать другим (wing 2). Это создаёт внутреннее напряжение: критика несовершенства мира vs. сострадание к людям.

7-Step Algorithm: Enneagram → Anchors

  1. Определите core fear → maps to GHOST
  2. Identify desire → maps to WANT
  3. Map stress behavior → generates FLAW behavior
  4. Map growth direction → hints at NEED
  5. Select 1-2 OCEAN poles that support the type
  6. Generate 2-3 anchors from stress/growth behavior
  7. Write OCEAN string in Description + Enneagram notation

→ Крылья (wing variants), интеграция/дезинтеграция, полный разбор типов → Layer 3

MBTI — Когнитивные функции

MBTI (Myers-Briggs Type Indicator) — система типов личности, основанная на когнитивных функциях. В контексте карточки персонажа MBTI помогает определить стиль мышления и восприятия.

Связь с SPINE:
MBTI определяет КАК персонаж обрабатывает информацию и принимает решения.
Enneagram определяет ПОЧЕМУ персонаж действует так.
OCEAN определяет КАКИЕ поведенческие паттерны наиболее выражены.

Формат в карточке

MBTI: INTJ. Функции: Ni→Te→Fi→Se.

Полный пример SPINE

Геральт из Ривии (The Witcher 3)

WANT: Сохранить нейтралитет ведьмака — избегать политики, зарабатывать на монстрах, избегать привязанностей.

NEED: Принять роль отца для Цири, понять, что истинная сила — в связях с близкими, а не в изоляции.

FLAW: Цинизм как щит. Отталкивает тех, кто пытается сблизиться, потом жалеет об этом.

LIE: "Если я никого не впущу — никто не сможет меня ранить. Нейтралитет = безопасность."

GHOST: Бледная Кобыла. Потеря Йеннифэр, ложные обвинения, изгнание из Каэр Морхена в юности.

Проверка SPINE:

  1. WANT и NEED в напряжении?
  2. FLAW конкретен (наблюдаемое действие)?
  3. LIE объясняет, почему FLAW "работает"?
  4. GHOST объясняет, откуда FLAW и LIE?
  5. FLAW имеет минимум 1 якорь?
