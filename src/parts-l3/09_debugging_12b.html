12B-14B Debugging

Контекст: Модели 12B-14B (Qwen 2.5 14B, Gemma 3 12B, Llama 4 Scout 17B, Mistral 12B) имеют ограниченный контекст и меньшую способность следовать сложным инструкциям.

Таблица симптомов 12B-14B

СимптомПричинаРешение
Голос "плывёт" Voice Leak в Description Удали стилистические директивы из Description. Голос только в Examples. → §R2
Персонаж OOC Temperature > 0.85 Снизь temperature до 0.6-0.8
Годмодинг Нет анти-годмодинга Добавь блок в System Prompt → §5.0.1
Повторяет фразы Token bloat Сократи Description до <600 токенов
Игнорирует якоря Нет Price Добавь Price к критичным якорям
Забывает контекст Контекст ≤4K Примени 4K-Fallback Protocol (ниже)
Теряет стиль речи presence_penalty > 0 Установи presence_penalty = 0.0 (зафиксируй!)
"Умный" персонаж Голос в Description Удали любые описания речи. Только демонстрация в Examples.
Формат дрейфует Нет Format Lock Добавь Format Lock в конец SP
Смешивает персонажей Multi-char без маркеров Добавь явные маркеры [ИМЯ] в Examples

Рекомендуемые параметры для 12B-14B

temperature = 0.6-0.8      # НЕ выше 0.85!
top_p = 0.9
min_p = 0.05
repetition_penalty = 1.05   # НЕ выше 1.10!
top_k = 50
presence_penalty = 0.0      # ЗАФИКСИРОВАНО! Критично!

⚠️ Presence Penalty = 0.0 КРИТИЧНО для 12B

Значения > 0 "убивают" речевые тики и уникальный голос. Это необратимо для малых моделей.

4K-Fallback Protocol

Для контекста ≤4K токенов:

БлокОбычный4K Fallback
System Prompt~100~50 (минимум)
Description~500~250 (только якоря)
Examples~300~150 (2 примера)
Greeting~80~50
Итого~980~500

Минимальный System Prompt для 4K:

Ты — [ИМЯ].
3-е лицо, настоящее время.
Не пиши за {{user}}.

Минимальное Description для 4K:

[ИМЯ] — [роль]. [2 ключевые черты].

• [Триггер] → [Действие] → Цена: [реакция]

Voice Isolation для 12B

Критичное правило: На 12B голос в Description = гарантированный дрейф.

Правило 12B:

  • Description: ТОЛЬКО факты и якоря
  • Examples: ВСЯ стилистика
  • Никаких "говорит саркастично" в Description

Почему это критично для 12B:

  • 12B хуже следует инструкциям → стилистика в Description игнорируется
  • 12B сильнее копирует паттерны → Examples = 85% влияния
  • При конфликте Description vs Examples → Examples побеждает

Диагностика по шагам

  1. Проверь Temperature — должна быть 0.6-0.8
  2. Проверь Presence Penalty — должна быть 0.0
  3. Проверь Description — нет ли голоса?
  4. Проверь Examples — есть минимум 2?
  5. Проверь токен-бюджет — Description ≤ 600?
  6. Проверь Format Lock — есть в конце SP?

Если все параметры ОК, но персонаж "плывёт":

→ Перепиши Examples с более ярко выраженным голосом. 12B учится через копирование.
