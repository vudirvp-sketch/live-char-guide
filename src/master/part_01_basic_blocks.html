Базовые блоки карточки

Карточка персонажа — это документ, который задаёт поведение ИИ-модели при ролевой игре. Но думать о ней нужно как о поведенческом движке: каждый элемент должен производить наблюдаемое действие, а не украшать описание. Карточка состоит из четырёх блоков, каждый из которых выполняет свою функцию.

Вы на Глубоком слое (L2). Этот раздел — краткий обзор базовых блоков. Порядок чтения L2: ① FLAW-linked якоря (Part 2) → ② Examples L2 (Part 3) → ③ SPINE (Part 4) → ④ OCEAN/Enneagram (Part 5) → ⑤ L2 Quickstart (Part 5).

💡 Хотите сразу к делу? Пошаговый конвейер сборки L2-карточки за 30 минут → L2 Quickstart. Рекомендуется сначала прочитать Parts 2–5 для понимания инструментов.

5 базовых правил

Эти правила обязательны для всех карточек, независимо от слоя. Нарушение любого из них приводит к ухудшению качества генерации.

📌 Фундаментальный принцип: LLM = Pattern Matcher, не Исполнитель правил. Директива в SP без паттерна в Examples нестабильна на моделях ≤14B (~40–60% следование директивам). Поэтому Examples — главный инструмент, а Description — логика поведения.

#ПравилоПочему важноPattern Matcher
1OCEAN: 1-2 экстремальных полюса — значения <30 или >70Больше 2 — внутренняя противоречивость. 0 — персонаж забываемый.3+ противоречивых полюса → модель «осциллирует» между паттернами
2Цена обязательна — каждый FLAW (Дефект)/GHOST якорь имеет физическую реакциюБез цены модель не показывает уязвимость.Без Price якорь = инструкция без паттерна → ≤60% следование директивам
3Голос задаётся ТОЛЬКО в Examples и Greeting, никогда в DescriptionМодель игнорирует инструкции о голосе, но копирует паттерны из Examples.Description = логика (~0% следование голосу на 12B); Examples = паттерн (~85% следование директивам)
4GHOST только в Description: GHOST (событие прошлого персонажа) — только в Description, никогда в System PromptПсихология в SP ломает поведение модели.SP = инструкции (модель игнорирует); Description = логика поведения
5Формат якоря: Триггер → Действие → ЦенаКаждый элемент обязательный. Пропуск — потеря качества.Каждый элемент = паттерн для копирования. Пропущенный элемент = слепая зона

Токен-бюджет

Сборка карточки — это конвейер, где каждый блок получает свой токен-бюджет. Превышение бюджета любого блока вытесняет другие, и модель теряет контекст. Таблица ниже показывает бюджет для L2-карточки (SP + Description + Examples + Greeting). Дополнительные блоки (Tone Frame, Format Lock, AN, Lorebook) описаны в Part 7.

Лимиты по блокам

Каждый блок имеет три уровня бюджета: минимальный (4K), стандартный (8K+) и максимальный (16K+). Превышение максимального значения — признак раздувания, ведущий к потере контекста.

📌 Превышение бюджета карточки → модель теряет контекст → персонаж «плывёт». Соблюдайте бюджет своего слоя. Если блок не помещается — сокращайте, а не раздувайте. Дополнительные блоки (AN, Lorebook, OCEAN, SPINE) доступны на более глубоких слоях — см. Следующие шаги.

Что дальше? Инструменты для углубления

Вы освоили минимальный слой L1 — этого достаточно для создания рабочей карточки. Если вы хотите углубить персонажа, три инструмента открывают новые возможности на более глубоких слоях.

ИнструментЧто даётДоступен на
SPINE (WANT/NEED/FLAW)Внутренняя логика персонажа — почему он действует именно так → SPINEСлой 2
OCEAN / EnneagramПсихологический профиль — экстремальные черты, которые делают персонажа уникальным → OCEANСлой 2
GHOST / LIE / CoTСкрытые противоречия и цепочка рассуждений — персонаж, который живёт в конфликте с собой → GHOSTСлой 3

💡 Совет: Все инструменты опциональны. L1-карточка уже работает — углубление добавляет глубину, но не является обязательным.

Глубокий слой (L2)

SPINE — фреймворк мотивации персонажа (WANT/NEED/FLAW), который связывает внутренний конфликт с поведением через якоря. Полный разбор → Part 4 Якоря перестают быть случайным набором правил и получают причину. OCEAN (Big Five) — инструмент проверки внутренней консистентности. Enneagram — модель из 9 типов. CORE DIRECTIVES — единая система из 5 директив для SP. Также: Author's Note, Lorebook, Format Lock, анти-паттерны.

  • Критерии перехода: карточка стабильна на L1, 2 Examples дают узнаваемый голос, 3–5 якорей покрывают основные ситуации
  • Время создания: ~30 минут (вместо ~15 на L1)
  • Токены: 800–1500 (вместо 400–800 на L1)

Экспертный слой (L3)

Экспертный слой добавляет LIE (ложную установку) и GHOST (событие прошлого), Chain of Thought (CoT), XML-структуру Description и расширенные анти-паттерны.

  • Критерии перехода: SPINE консистентен, 3 Examples показывают разные эмоции, FLAW-linked якоря работают, используете 32B+ или API модель
  • Время создания: ~60 минут
  • Токены: 1500+

Топ-3 критичные ошибки

Три самые частые ошибки начинающих, которые сильнее всего ломают качество карточки. Каждая разбирается подробно: симптом → причина → пошаговое исправление → пример до/после. Для краткого справочника по 5 проблемам см. Топ-5 проблем.

Ошибка 1: Персонаж пишет за {{user}}

Симптом: Модель описывает действия, мысли и реплики {{user}}. Диалог превращается в монолог от двух лиц.

Причина: В System Prompt нет анти-годмодинга.

Исправление: Шаблон SP → Part 7 | AP-6: Нет анти-годмодинга → Part 8

Ошибка 2: Голос не соответствует описанию

Симптом: Вы описали персонажа как «лаконичного и холодного», но модель отвечает многословно и эмоционально.

Причина: Голос помещён в Description вместо Examples — правило #3. Модель — Pattern Matcher: она копирует паттерны из Examples, но почти не следует стилистическим директивам в Description.

Исправление: Полный разбор Voice Isolation → Part 3

Ошибка 3: Персонаж «слишком идеальный»

Симптом: Персонаж не проявляет уязвимость, не ошибается, реагирует «правильно» на все стимулы.

Причина: Якоря не имеют цены (Price). Без физической реакции якорь = инструкция без паттерна.

Исправление: Критерии качества якоря → Part 2
