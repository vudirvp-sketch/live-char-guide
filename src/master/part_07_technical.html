Техническая реализация

System Prompt (SP) — блок инструкций, задающий рамки поведения модели.

Обязательные элементы SP

  • Identity: "You are {{char}}"
  • Anti-godmoding: "Never speak or act for {{user}}"
  • Character constraints: Только L1 — минимальные

Шаблон SP (L1)

You are {{char}}. Never speak or act for {{user}}.

Rules:
- Stay in character at all times
- Use the dialogue format from examples
- Never break the fourth wall

Шаблон SP (L2)

You are {{char}}. Never speak or act for {{user}}.

Identity:
- Name: {{name}}
- Role: {{role}}

Constraints:
- Stay in character at all times
- Use the dialogue format from examples
- Never break the fourth wall
- Never apologize unless character would

Format Lock:
- Dialogue: *action* "speech"
- Internal thoughts: [INTERNAL]...[/INTERNAL] (L3 only)

Format Lock

Format Lock — фиксация формата диалога в конце SP. Предотвращает дрейф разметки.

3 системы разметки диалога

СистемаСинтаксисДля когоПример
A*action* dialogueУниверсальная, L1/L2 рекомендована*sighs* "Fine."
B— dialogueМинималистичная, EN модели— Fine.
C[action] dialogue12B и API, явная структура[sighs] Fine.

Format Lock инструкция

Format Lock:
- Dialogue: *action* "speech"
- Thoughts: *thinks: {{thought}}*
- Never change this format

📌 Правило: Выберите одну систему и зафиксируйте её в Format Lock. Не смешивайте системы в одной карточке.

Author's Note (AN)

Author's Note — динамический контекст, вставляемый перед последними сообщениями. Задаёт сцену, настроение, мотивацию.

Параметры AN

ПараметрРекомендация
Позиция3-5 сообщений от конца
Длина100-200 токенов
ЧастотаКаждые 5-10 сообщений

Пример AN

[Author's Note: Сцена — ночь, дождь. Елена уставшая после долгого дня. Она не доверяет {{user}}, но устала слишком сильно, чтобы защищаться.]

Structured Inject

Structured Inject — техника использования XML-тегов в AN для динамической мотивации.

Правила Structured Inject

  • Не дублируйте SP — добавляйте контекст
  • Не конфликтуйте с Description
  • Задавайте мотивацию, не инструкции

Пример Structured Inject

[Author's Note:
<scene_context>Ночь, библиотека закрыта, только они двое</scene_context>
<emotional_state>Усталость + осторожное любопытство</emotional_state>
<hidden_agenda>Елена хочет узнать, зачем {{user}} пришёл</hidden_agenda>
]

Lorebook

Lorebook — база знаний с триггерами. Используется для GHOST-связанных фактов.

Структура записи Lorebook

ПолеОписаниеПример
KeyТриггер-слово"предательство", "редактор"
ContentФакт для вставки"История о коррупции была украдена редактором..."
PositionКуда вставлятьAuthor's Note (char)
DepthКак давно искать5 сообщений

Правила Lorebook

  • GHOST-факты = кандидаты для Lorebook
  • Не дублируйте Description
  • Триггеры = конкретные слова, не абстракции
Расширенный XML-теги → Экспертный слой→
